# Security Policy

## Supported Versions

Security fixes are applied to the latest project state only.

| Version | Supported |
| --- | --- |
| `main` | Yes |
| Older commits, branches, or tags | No |

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

Use one of these private channels instead:

1. GitHub private vulnerability reporting for this repository, if it is enabled.
2. Private contact through the repository owner's GitHub profile: [`DrEden33773`](https://github.com/DrEden33773).

Please include:

- A clear description of the issue
- Reproduction steps or a minimal proof of concept
- Expected impact
- Any known mitigations or workarounds

## Disclosure Expectations

- We will try to acknowledge receipt within 7 days.
- We may ask for extra reproduction details before confirming severity.
- Please avoid public disclosure until a fix or mitigation is available.

## Non-Security Issues

For normal bugs, feature requests, and documentation issues, use the public issue tracker once the repository is public.
